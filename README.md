# Kintone モバイル版 画像タップ拡大表示

Kintone の**モバイル版**で、**画像をタップするだけで全画面拡大表示**するカスタマイズです。
PC（Web）版は Kintone 標準の画像ビューアが既に同等機能を備えているため、**本カスタマイズは
モバイル版でのみ動作**し、PCには一切干渉しません（標準ビューアをそのまま使います）。

対応領域（モバイル）:

- お知らせ掲示板
- スペースのスレッド・コメント
- レコード詳細の**リッチテキスト（本文編集）内の画像**

拡大表示には [PhotoSwipe v5](https://photoswipe.com/)（MITライセンス）を使用し、
**ピンチズーム・スワイプによる複数枚送り**に対応しています。

---

## 設計方針

- **モバイル限定**: URL が `/k/m/` 配下のときのみ動作（PCでは即座に処理を打ち切り、標準ビューアを尊重）。
- **document委譲＋祖先フォールバック**: 領域ごとの固定クラスに依存せず、コンテンツルート
  （モバイルは `#container`）配下の画像クリックを1つのハンドラで受けます。
  グループ（スワイプ送り単位）は「リッチテキスト → スレッド/コメント → お知らせ記事ブロック」の
  優先順で最も近い祖先を採用し、**該当が無ければ画像の親要素を1グループ**として必ず開きます。
  → これにより、専用クラスを持たない**お知らせ掲示板でも確実に動作**します。
- **アップロード＋バンドル1ファイル化**: 外部CDNに依存せず、PhotoSwipe本体と自作コードを
  1つのJSにバンドル。閉域環境・厳しいセキュリティポリシーでも動作します。

## ディレクトリ構成

```
.
├── src/
│   ├── index.js              # エントリ（モバイルガード → 有効化）
│   ├── core.js               # PhotoSwipe初期化・委譲・グループ解決・除外
│   └── config.js             # セレクタ・閾値（壊れたらここを調整）
├── build.mjs                 # esbuildビルドスクリプト
├── dist/                     # ★Kintoneにアップロードする成果物
│   ├── kintone-image-viewer.bundle.js
│   └── photoswipe.css
└── test/
    ├── local-test.html       # モバイル想定の動作確認（掲示板/スレッド/リッチテキスト/除外）
    └── guard-test.html       # 非モバイルで動作しないこと（PC保護）の確認
```

## 適用手順（Kintoneへの登録）

> **全体カスタマイズの「スマートフォン用」スロットに登録するだけ**で完了します。
> 各アプリへの個別登録は不要です（全体カスタマイズはモバイルのポータル・スペース・
> レコード画面に行き渡ります）。**PC用スロットには登録しないでください**（標準ビューアと
> 競合させないため）。

### 手順

1. 右上の **歯車アイコン → 「kintoneシステム管理」** を開く
   （※「cybozu.com共通管理」ではなく **kintoneシステム管理** です。システム管理者権限が必要）
2. **「JavaScript / CSSでカスタマイズ」** を開く
3. **「スマートフォン用のJavaScriptファイル」** に `dist/kintone-image-viewer.bundle.js` をアップロード
4. **「スマートフォン用のCSSファイル」** に `dist/photoswipe.css` をアップロード
5. **「PC用」スロットには何も追加しない**（PCは標準ビューアのまま）
6. 適用対象は **「すべてのユーザーに適用」** を選択
7. **「保存」**

### 動作確認

- スマートフォンの Kintone アプリ／モバイルブラウザ（`/k/m/`）で、
  - **お知らせ掲示板**の画像をタップ → 全画面拡大
  - **スペースのスレッド／コメント**の画像をタップ → 拡大＋スワイプ送り
  - **レコードのリッチテキスト**内画像をタップ → 拡大
- PC版では従来どおり **Kintone標準のビューア**が開く（本カスタマイズは動作しない）

> スマホ実機が無い場合は、PC Chrome の開発者ツールでURLを `https://（あなたのサブドメイン）.cybozu.com/k/m/`
> にし、デバイスツールバー（Ctrl+Shift+M / ⌘+Shift+M）でモバイル表示にして確認できます。

## 添付画像・リッチテキスト画像の扱い（重要）

Kintoneの添付画像は `src` が **`w=NNN` のサムネイル**、原寸URLが **`data-original`** に入る場合があります
（いずれも内部の認証付き `download.do` URL）。ただし**画像の種類で挙動が異なる**ことが実測で判明しています。

| 種類 | クラスの特徴 | 有効なURL |
|---|---|---|
| 掲示板など | `gaia-ui-slideshow-thumbnail` | **`data-original`（原寸）が有効** |
| レコードのリッチテキスト | **`user-token`**-…-slideshow-thumbnail | **サムネ `src`（`w=NNN`）のみ有効**。`data-original` や `w` 除去・拡大は **404** |

そのため `resolveUrls`（[src/core.js](src/core.js)）は次の方針でURLを解決します。

1. クラスに **`user-token` を含む**（＝レコードのリッチテキスト）→ **サムネ `src` をそのまま使用**
2. それ以外で `data-original` があれば **原寸**を使用
3. `<a href>` の画像リンクがあればそれ
4. いずれも無ければ `src`

さらに**保険**として、選んだURLの読み込みに失敗した場合は PhotoSwipe の `loadError` を捕捉し、
**必ず読めるサムネ `src` で自動再読込**します（「画像を読み込めない」表示を構造的に防止）。
URLは事前収集せず**タップ時点でDOMから参照**します。

> 制約：レコードのリッチテキスト画像は、上記の署名URLの仕組み上モバイルでは**サムネ解像度（≈250px）が上限**です。
> 掲示板は `data-original` により原寸表示されます。

### 除外設計の注意（スライドショー）
掲示板/リッチテキストの添付サムネは `gaia-ui-slideshow-thumbnail` というクラスを持ちます。
ポータル先頭の装飾スライドショー（`cybozu-ui-slideshow`）だけを除外し、**添付サムネは除外しない**よう
`config.js` の `EXCLUDE_ZONES` を `[class*="cybozu-ui-slideshow"]` に限定しています
（`[class*="slideshow"]` のように広く除外すると添付画像まで拾えなくなるため注意）。

> なおモバイル版ではKintone標準のスライドショー処理が `TypeError` で失敗することがありますが、
> 本カスタマイズが capture フェーズ＋`stopPropagation` で先取りするため、標準処理は抑止され当方ビューアで開きます。

## セレクタが効かない場合（メンテナンス）

KintoneのDOM構造（class名）は非公開仕様で、アップデートにより変わることがあります。
拡大表示が効かなくなった場合は、本体（`core.js`）ではなく `src/config.js` を調整してください。

- `CONTENT_ROOTS` … モバイルのコンテンツルート（既定 `#container`）
- `GROUP_CANDIDATES` … スワイプ送り単位の優先候補（上から順に探索）
- `EXCLUDE_ZONES` / `EXCLUDE_IMG` … 拡大対象から外す領域・画像
- `iconMinSize` … アイコン除外のサイズ閾値

調整後は `npm run build` で再ビルドし、`dist/` の2ファイルを再アップロードします。

## ローカル動作確認

```bash
npm run build
python3 -m http.server 8766
# モバイル想定 : http://localhost:8766/test/local-test.html
# PC保護の確認 : http://localhost:8766/test/guard-test.html
```

`local-test.html` は `window.__KIV_FORCE__ = true` でモバイルガードを一時的に解除して
動作を確認できます（本番では不要）。`guard-test.html` は force を設定せず、非モバイルでは
何も起きないこと（PCで標準ビューアを尊重すること）を確認できます。

検証済みの挙動:
- お知らせ掲示板（専用classなし → 親ブロックで `1 / 2` グループ化）
- スレッドコメント（専用class → `1 / 3` グループ化）
- リッチテキスト（`1 / 1`）
- ヘッダー/ナビ/アイコンは拡大対象外
- 非モバイルパスでは一切バインドされない

## 注意事項

- `npm audit` の `esbuild` 警告は**ビルド時の開発サーバー**に関するもので、Kintoneへ配布する
  `dist/` のバンドルには含まれません（実害なし）。
- 既存のモバイル側カスタマイズと画像クリックが競合する場合に備え、本実装は capture フェーズ＋
  `stopPropagation` で優先動作します。
- 実機（iOS Safari / Android Chrome）での最終確認を推奨します。
